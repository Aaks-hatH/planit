FAQ

Frequently asked questions

Answers to the most common questions from organisers, staff, and guests — covering everything from account basics to advanced check-in scenarios.

General
Is PlanIt free to use?

Yes — completely free. There are no paid tiers, no per-event fees, no feature paywalls, and no credit card required. PlanIt is proprietary freeware operated as a hosted service. You use the official app at planitapp.onrender.com.

Do guests need to create an account or download an app?

No. Guests receive a unique invite link and open it in any browser. There is no sign-up, no password, and no app to install. The invite page works on iOS Safari, Android Chrome, and desktop browsers without any additional setup.

Can I self-host PlanIt or run my own instance?

No. PlanIt is proprietary software. The source code is published for transparency and audit purposes only. Self-hosting, forking, and redistribution are not permitted under the licence. All events must run on the official PlanIt infrastructure.

How long is event data kept?

All event data is automatically and permanently deleted seven days after the event date. This includes every message, file, guest record, check-in log, seating map, and the event itself. A countdown warning banner appears in the organiser dashboard as the deadline approaches. Deletion is irreversible — export anything you need before the window closes. Note: Table Service mode is exempt from this policy; restaurant venues persist indefinitely.

What is the difference between Standard, Enterprise, and Table Service mode?

Standard mode is a full planning workspace for your team: chat, tasks, polls, notes, announcements, budgets, files, agenda, and countdown timer. No guest list or check-in system.

Enterprise mode includes everything in Standard, plus a complete guest management and QR check-in system with anti-fraud middleware, seating maps, walkie-talkie, and live analytics.

Table Service mode is an entirely separate interface for restaurants and hospitality venues: a live floor plan, walk-in waitlist, QR reservations, and per-table management. No planning workspace. Data does not auto-delete.

The mode is chosen at event creation and cannot be changed afterward.

Can I manage multiple events from one account?

PlanIt uses a no-account model — each event workspace is independent. There is no central dashboard that aggregates multiple events. Each event has its own URL, its own organiser password, and its own team. If you run multiple events, you manage each one separately via its own workspace link.

How do I delete an event before the 7-day window?

From the event workspace, open Settings (gear icon in the header) → scroll to the danger zone → click Delete Event. You will be prompted to confirm. Deletion is immediate and permanent. This option is only visible to the organiser.

Is there a mobile app?

PlanIt is a progressive web app (PWA). There is no native app in the App Store or Google Play. However, you can install PlanIt to your home screen from any modern mobile browser. On iOS: tap the Share button in Safari → Add to Home Screen. On Android: tap the three-dot menu in Chrome → Add to Home Screen or Install App. Once installed, it launches in a standalone window without browser chrome.

Creating & Managing Events
Can I change the event mode after creation?

No. Standard, Enterprise, and Table Service mode are architectural — the database schema, routes, and UI differ fundamentally between modes. If you chose the wrong mode, create a new event with the correct mode. There is no migration tool.

Can I change the event URL slug after creation?

No. The slug is set at creation and becomes the persistent identifier for the event. Changing it would break any invite links already sent to guests or team members. Choose your slug carefully before sharing the event link.

I forgot my organiser password. Can I reset it?

There is no self-service password reset. PlanIt stores no email address, so there is nothing to send a reset link to. If you lose your organiser password, contact support via /support. Recovery may be possible if you can verify ownership of the event, but it is not guaranteed. Always save your password in a password manager before sharing your event link.

Can I edit the event name, date, or location after creation?

Yes. Open the event settings panel (gear icon) as organiser and edit any field. Changes propagate immediately — the invite pages update within seconds, and the countdown recalculates from the new date/time. Note that changing the event date resets the 7-day deletion window.

Can I password-protect my event workspace?

Yes. Set an event password at creation time, or add one later from Settings. Anyone who opens the workspace URL — guests, team members, or others — must enter this password before they can view the event. This is separate from the organiser (account) password. Leave it blank for an open, password-free workspace.

Check-in & QR Scanning
A guest's QR code won't scan. What do I do?

Use the Manual check-in option: tap the keyboard icon in the scanner toolbar and type the 8-character invite code directly. Common scan failures:

  • Screen brightness too low — ask the guest to turn it up to maximum
  • Screen glare — tilt the phone slightly away from overhead lights
  • The QR image has been cropped, zoomed, or distorted in a screenshot
  • Camera permission not granted — check browser settings and allow camera access
  • Dark mode inversion — some apps invert the QR colours which breaks scanning
A guest was accidentally checked in. Can we undo it?

From the check-in dashboard guest list, find the guest and click the undo/reset icon on their row (visible to organisers). This marks them as not-yet-checked-in so their QR becomes scannable again. The undo is logged in the audit trail.

The check-in page shows "already checked in" but it was a mistake. How do we admit them?

On the denial screen, tap Manager Override. An organiser enters their account password to approve entry. The override is logged in the activity log with the approving organiser's identity and a timestamp.

Can we run multiple check-in lanes with different devices?

Yes — open the check-in page on as many devices as needed. Each device logs in with the staff credentials (or organiser password). Check-ins sync in real-time: if a guest is admitted on device A, device B will immediately show "already checked in" if the same code is scanned. There is no per-device setup or pairing required.

The internet dropped at our venue. Does check-in still work?

Yes — if the check-in page was fully loaded while online. PlanIt caches the guest list into the browser when the page loads. Offline scans are queued locally and automatically synced to the server when connectivity returns. Important: load the check-in page and let it fully initialise before you lose signal. If the page was never loaded with an internet connection, the cache won't exist.

How do I check in a guest who doesn't have their phone with them?

Use Manager Override: tap the manual override button, authenticate with the organiser password, and search for the guest by name. Select them and tap Admit. This is logged as a manual override in the audit log.

What does the anti-fraud system actually do?

PlanIt's anti-fraud middleware runs on every check-in attempt and performs several checks:

  • Duplicate detection — fingerprints each invite and warns if a matching identity is already admitted
  • Reentrancy locking — prevents two staff devices processing the same QR simultaneously (race-condition protection)
  • Rapid-scan detection — flags if the same code is scanned 3+ times within 10 seconds
  • Multi-device detection — logs if the same code is scanned from significantly different locations within a short window
  • Trust score — each invite starts at 100; security events reduce it; you can configure an auto-block threshold
  • Time window enforcement — by default, check-in only works from 2 hours before the event to 30 minutes after it starts

Warnings are shown to staff but do not automatically block admission unless the auto-block setting is explicitly enabled. Staff always make the final call.

What is the check-in time window?

By default, the check-in system accepts scans starting 2 hours before the event and up to 30 minutes after it starts. Scans outside this window are rejected with a message explaining when the window opens or that it has closed. Organisers can extend or disable the window from the check-in settings panel.

Guests & Invites
A guest lost their invite link. Can I resend it?

Yes. Find the guest in the check-in guest list, click the share/copy icon on their row, and copy the invite link again. The link is permanent — it doesn't change unless you delete and recreate the guest record.

Can a guest bring additional people not on the invite?

Each invite records an expected group size (adults + children). Staff see this on the boarding pass at check-in. If a guest arrives with a different count, the organiser can edit the invite to update the number, or staff can note the actual headcount. The system tracks expected vs actual separately.

How do I send invites — does PlanIt email guests automatically?

PlanIt does not send emails on your behalf. You send invite links yourself via whatever channel you prefer — email, WhatsApp, SMS, Slack, printed QR codes, etc. Copy the invite URL from the guest list and send it directly. You retain full control over how and when guests receive their invites.

Can I import guests in bulk?

Yes. Use Import CSV from the check-in toolbar. Download the template first to get the correct column headers: guestName, guestEmail, guestRole, adults, children, securityPin, notes. Only guestName and guestEmail are required. Roles are GUEST, VIP, or SPEAKER.

What is a security PIN and when should I use one?

A security PIN (up to 6 characters) is an optional second factor for individual guests. When set, staff are prompted to verify the PIN at check-in before admitting the guest. Use it for high-value guests, VIPs, or situations where you want extra confirmation that the right person is presenting the code. The PIN is never shown on the invite page — only on the boarding pass for staff to compare against what the guest provides verbally.

How do I print badges?

Each guest has a badge page at /badge/INVITECODE. Open it and print from your browser (File → Print). Badges are formatted for A6 paper. For large guest lists, open tabs for each guest in bulk and print in sequence, or print from a list of badge URLs. There is no bulk-print queue in the current version.

Can guests add the event to their Apple or Google Wallet?

Yes. Every guest's invite page includes an Add to Wallet button. On iOS it adds to Apple Wallet as a pass. On Android it adds to Google Wallet. Wallet cards display the guest name, event name, date, location, QR code, and table assignment. They update dynamically if the organiser changes the event details.

What is the difference between the invite page, boarding pass, badge, and wallet card?
  • Invite page (/invite/CODE) — the full guest-facing page with event details, QR code, table assignment, and wallet/badge download options. This is what guests open from their link.
  • Boarding pass (/card/CODE) — a compact card view optimised for presenting at the door. Shows QR code, name, role, group size, and notes.
  • Badge (/badge/CODE) — a printable A6 layout for name badges. Includes name, role, event, QR code.
  • Wallet card — a digital pass stored in Apple Wallet or Google Wallet. Tap-to-show on lock screen, no internet needed at the venue.
Seating
Can guests see their table assignment before the event?

Yes. Once you assign a guest to a table in the seating map, that table label appears on their invite page and wallet card immediately. If you prefer to reveal seating on the day, simply assign tables the morning of the event — or keep seating as staff-only information by not publishing it to the invite page.

Can I move guests between tables?

Yes. Edit the guest record (edit icon in the guest list or click the guest in the seating map) and change the table assignment. The seating map, guest list, and invite page all update live.

What types of table shapes are supported?

The seating map editor supports: round tables, rectangular tables, long banquet tables, VIP booths, bar areas, stage/performance areas, dance floors, and free-label zones. Each object can be labelled, resized, and repositioned on the canvas.

Can staff see the seating map during check-in?

Yes. The seating map button is visible to all logged-in staff from the check-in page. Staff get a read-only view that shows table positions and names. Organisers get the full editor. On the boarding pass screen, a guest's table assignment is shown prominently so staff can direct them without opening the map.

Can I export the seating chart?

There is no one-click export yet. Screenshot the canvas, or compile a table-by-table list by filtering the guest list by table name. You can also export the guest list CSV and sort by table column.

Planning Tools
Who can post announcements?

Only users logged in as organiser can post to the Bulletin (announcements) tab. All other participants can read announcements but not create them. This makes the Bulletin a one-way broadcast channel from the organiser to the team.

Can I change my vote in a poll?

No — votes are final once cast. There is no vote-change mechanism. If this is a concern, create a new poll and close (delete) the old one.

Is there a limit on file upload size?

Individual file uploads are limited to the Cloudinary upload limit for the plan in use (typically up to 100 MB per file). The Files tab limits 20 uploads per hour per IP address. All uploaded files are deleted when the event data is purged at the 7-day mark.

Are notes, tasks, and polls visible to guests?

No. The workspace planning tools (chat, tasks, polls, notes, announcements, budget, files, agenda) are visible only to participants who have joined the workspace by opening the event link. Guests who only have their personal invite link (/invite/CODE) see only their invite page — they never see the workspace.

Can I export the .ics calendar file after the event is created?

Yes. Open the Share tab in the workspace and click Add to Calendar. This downloads a standard .ics file compatible with Apple Calendar, Google Calendar, Outlook, and any other iCalendar-compatible app. The file contains the event name, date, time, location, and workspace URL.

Walkie-Talkie
Does the walkie-talkie work on all devices?

It requires a browser that supports WebRTC and the Web Audio API. This includes Chrome (Android and desktop), Safari 15+ (iOS 15+), and Edge. Firefox is supported with some latency variance. Internet Explorer and very old Safari versions do not support WebRTC and cannot use the walkie-talkie.

Can guests use the walkie-talkie?

No. The walkie-talkie is available exclusively to logged-in staff and organiser sessions on the check-in page. Guests with only their invite link cannot access it.

How many people can be on the walkie-talkie channel at once?

All logged-in staff/organiser sessions are automatically on the channel — there is no limit or separate join step. Only one person can transmit at a time (push-to-talk). All others listen simultaneously. The channel is always event-scoped, meaning staff from different events never hear each other.

My push-to-talk button isn't working. What should I check?
  • Make sure microphone permission is granted in the browser (look for a mic icon in the address bar)
  • Check that no other app is exclusively holding the microphone
  • On iOS, go to Settings → Safari → Microphone and set to Allow
  • Reload the check-in page and try again
  • The button requires a user gesture to initialise on some browsers — tap elsewhere on the page first, then try the walkie-talkie button
Table Service Mode
Is Table Service mode free?

Yes. Table Service mode is part of the same free PlanIt platform. No subscription, no per-seat fees.

Does the Table Service data get deleted after 7 days like event data?

No. Table Service venues are exempt from the 7-day deletion policy. Restaurant floor data persists indefinitely because venues operate continuously, not in discrete event windows. You can manually clear reservations or reset the floor plan at any time from the management settings.

Can customers make QR reservations themselves?

Yes. The venue's QR code leads to a reservation form where customers enter their name, party size, and contact number. The reservation appears in the waitlist. Staff can confirm it, assign it to a table, and the customer's QR code becomes a digital "arrival pass". Reservation QR codes expire after a configurable window (default 30 minutes).

The camera scanner won't start. What should I check?
  1. Look for a camera or lock icon in the browser address bar and set access to Allow
  2. On iOS: Settings → Safari → Camera → Allow
  3. On Android: Settings → Apps → Chrome → Permissions → Camera → Allow
  4. Ensure no other app (video call, FaceTime) is currently using the camera
  5. Try a different browser — Chrome has the most consistent scanner support
  6. Reload the page — the scanner initialises at page load
I see "Service Crash" or an error loading the app. What does that mean?

PlanIt runs on Render's free-tier infrastructure. The backend server spins down after 15 minutes of inactivity and takes 30–60 seconds to cold-start when traffic returns. If you see a loading spinner or service error on first visit, wait 60 seconds and refresh. If the error persists beyond 2 minutes, check the status page at /status.

I found a bug. How do I report it?

Use the support form at /support. Include: what you were doing, what you expected, what actually happened, your browser name and version, and your device type. Screenshots or screen recordings are especially helpful. Reports are reviewed by the PlanIt team.
