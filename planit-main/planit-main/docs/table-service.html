Features

Table Service Mode

A live floor management system for restaurants, private dining rooms, and hospitality venues. Real-time table status, walk-in waitlists, QR reservations, and server coordination — no planning workspace, just the floor.

Table Service is a completely different mode

Selecting Table Service at event creation opens a different interface entirely. There is no planning workspace (chat, tasks, polls, notes, budget). The mode cannot be changed after creation. Venue data is never auto-deleted — it persists indefinitely.

Overview

Table Service mode is designed for venues that need to manage a live dining floor in real time — seeing which tables are occupied, who is next on the waitlist, how long each party has been seated, and which server is responsible for each table. Everything updates live across all staff devices via WebSocket.

The interface lives at /e/your-venue/floor and is separate from the standard event workspace URL.

The floor plan

The floor plan is a visual grid of your venue's tables. Each table shows:

  • Table number or name
  • Capacity (number of seats)
  • Current status (colour-coded)
  • Party name and size (when occupied)
  • Time seated (live elapsed timer)
  • Assigned server

Seating a party

  1. 1
    Click an available table

    Tap any green (available) table on the floor plan. A detail panel slides in from the right.

  2. 2
    Enter party details

    Fill in the party name, number of guests, any notes (dietary restrictions, special occasions), and optionally assign a server. Phone number is optional but useful for contact.

  3. 3
    Seat the party

    Click Seat Party. The table turns red on all connected staff devices simultaneously. A live elapsed timer starts.

  4. 4
    Mark as paid & clear

    When the party leaves, click the table and tap Clear Table. Choose whether to mark it Cleaning (grey) or immediately Available (green). The timer stops and the party details are archived in the session log.

Walk-in waitlist

The waitlist panel (accessible from the floor view) shows parties waiting for a table. Each entry shows the party name, size, time added, and an estimated wait time based on current floor occupancy and average turn times.

  1. 1
    Add a walk-in

    Click Add to Waitlist. Enter party name, size, and optionally a contact number. The party is added to the queue immediately.

  2. 2
    Seat from waitlist

    When a table opens, click the first matching party on the waitlist and select Seat Now. This removes them from the waitlist and opens the seating form pre-filled with their details.

  3. 3
    Remove from waitlist

    If a party leaves before being seated, click their entry and tap Remove. The list re-sorts automatically.

QR reservations

Your venue has a unique QR code that customers can scan to make a reservation. The QR code leads to a mobile-friendly reservation form where they enter:

  • Name
  • Party size
  • Contact phone number
  • Requested time (optional)
  • Special requests (optional)

Submitted reservations appear in the waitlist immediately. Staff can accept, reject, or assign them to specific tables. Reservation QR codes generated for the customer expire after a configurable window (default: 30 minutes) — if the customer arrives after expiry, they re-enter the waitlist as a walk-in.

Where to display your venue QR

Print the venue QR code and place it at:

  • The front-of-house host stand
  • On physical menus
  • On table tent cards
  • In your venue's window display
  • On your website or social media bio

Server assignment

Each table can be assigned to a named server. The floor plan view can be filtered by server to show only their section. This helps hosts coordinate floor coverage and ensures the right person is alerted when their section has a new party.

Occupancy overview

The header of the floor plan always shows:

  • Tables occupied / total tables
  • Guests seated / total capacity
  • Parties on the waitlist
  • Average time seated (across currently occupied tables)

Data retention

Table Service venues are exempt from the 7-day event data deletion policy. Floor plan configuration, server assignments, and venue settings persist indefinitely. Individual party session records (who was seated, when, at which table) are archived for reference but do not count toward any deletion window.

You can manually reset the floor plan or clear session history from the venue management settings at any time.

Multiple staff devices

Open the floor view on as many tablets or phones as you need. Every change — a table status update, a waitlist addition, a reservation confirmation — syncs instantly across all connected devices via Socket.IO WebSocket. There is no per-device setup. All staff see the same live floor at all times.

Best setup: tablet at host stand, phone in server pockets

A shared tablet at the host stand works well as the primary floor view. Servers with phones can check their section and update table status without walking back to the host stand. All views stay in sync automatically.
