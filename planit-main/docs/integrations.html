Integrations & QR

Using PlanIt with external tools

Embed QR codes, use media endpoints in emails and PDFs, and understand how email delivery works.

On this page
QR media endpoints Invite card endpoint Print badge endpoint Embedding in external pages Email sharing
These are user-facing media URLs, not a developer API

The endpoints described here are for organisers to use when sharing or printing invites — not for building third-party integrations or programmatic workflows. There is no public API key system.

QR media endpoints

Every invite has three media endpoints, each serving a different format of the invite's QR code and details. All endpoints are unauthenticated — the invite code itself is the access credential, the same as the invite page.

EndpointReturnsBest for
/qr/CODE SVG image (direct, no HTML wrapper) Embedding in emails, PDFs, other websites as a plain <img>
/card/CODE HTML page — mobile wallet card Sending to guests to screenshot and save
/badge/CODE HTML page — print-ready A6 badge Printing name badges before an event

Replace CODE with the invite code in all uppercase, e.g. AB12CD34.

Direct QR image — /qr/CODE

This endpoint serves the raw branded QR code as an SVG image, with no HTML page around it. It is a 301 redirect to the full SVG endpoint.

Use it anywhere you need to display the QR as an image:

<!-- In an HTML email -->
<img src="https://planitapp.onrender.com/qr/AB12CD34"
     width="200" height="200" alt="Entry QR Code" />
<!-- In an HTML page -->
<img src="https://planitapp.onrender.com/qr/AB12CD34"
     style="width:200px;height:auto" alt="QR" />
Always use height:auto

The QR SVG has a portrait aspect ratio (300 × 430). Forcing it into a square by setting both width and height will squash the QR modules and make it unreadable. Always set width only and let height scale automatically.

Invite card — /card/CODE

A full-screen mobile page styled like a boarding pass. Shows the QR code, guest name, event details, role, and table assignment against a dark background.

Share this URL with the guest directly — they open it on their phone, screenshot it, and present it at check-in. The card includes a share button that copies the URL.

The card is colour-coded by role: amber for VIP, purple for Speaker, neutral grey for standard guests.

Print badge — /badge/CODE

A print-optimised page sized for A6 paper (148 × 105 mm) and common label sheets. Open the URL in a browser, click Print Badge, and choose your paper size.

Print CSS automatically removes the on-screen button and instructions. The output is a clean badge with no browser chrome.

For bulk printing, open multiple badge URLs in separate tabs and print them in sequence. Each prints as a single page.

Embedding in external pages

You can embed a guest's QR code on any external page — a ticketing confirmation email, an event listing site, a printed PDF — by using the /qr/CODE image URL:

<!-- PDF-generating tool or custom confirmation email -->
<img src="https://planitapp.onrender.com/qr/AB12CD34"
     width="180" alt="Event entry QR code" />
<p>Present this at the entrance. Code: AB12CD34</p>

The image is served with a 24-hour cache header so it loads quickly on repeat views.

Email sharing

PlanIt does not send emails automatically. Email delivery is an optional action you take manually after creating an invite.

From the invite create dialog, click Send Invite Email. This opens a pre-filled draft in your default mail client with the guest's name, event name, and invite link in the body. You review and send it yourself.

For automated email delivery at scale, copy the invite links from your guest list and use your own email tool (Mailchimp, Brevo, etc.) to distribute them. The /qr/CODE image URL can be embedded directly in email templates for an inline QR code.
