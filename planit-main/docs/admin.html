Admin Panel

The command centre

System-wide oversight, analytics, marketing tools, and platform health — available to PlanIt platform administrators.

Platform administrators only

The admin panel at /admin is for PlanIt platform administrators — not for event organisers. Event organisers do not have access to the admin panel. If you are an organiser, you do not need this section.

Overview

The admin panel provides a global view of all events running on the PlanIt platform. It is accessed at /admin and requires a separate admin credential distinct from organiser accounts.

Events management

The events table shows all active and recent events across the platform with their organiser, participant count, invite count, check-in rate, and status. Administrators can:

  • View full event details and guest lists
  • Search and filter events by organiser, date, or status
  • Force-close or delete events in exceptional circumstances
  • Export event data for compliance purposes

Analytics

Platform-wide analytics aggregate data across all events and provide visibility into:

  • Total events created and active this period
  • Total guests invited and admitted across all events
  • Check-in rate trends over time
  • Most active organisers
  • Peak usage times

Analytics data is anonymised at the event level and does not expose individual guest information outside of its own event context.

Marketing tools

The marketing section provides tools for communicating with organisers and guests at scale. These tools comply with applicable anti-spam laws and operate only on contacts who have used PlanIt.

  • Announcements — publish platform-wide notices visible to all organisers
  • Email campaigns — send targeted emails to organiser segments via the configured email provider (Brevo or Mailjet)
  • Waitlist management — view and action the platform waitlist

System health

The status view shows real-time health for all platform components:

  • API server response times and error rates
  • Database connection status
  • Redis cache status
  • WebSocket server (mesh) connectivity
  • Uptime history with incident log

The public status page is available at /status and does not require admin credentials.

Bug reports

User-submitted bug reports from the /support page are routed to the admin panel for review. Each report includes the user's description, their browser and OS information, and a timestamp. Administrators can mark reports as resolved or escalate them.

Data retention

PlanIt automatically deletes all event data seven days after the event date. This includes messages, files, guest lists, check-in records, and the event itself. Organisers see a warning banner when their event is within seven days of deletion.

The retention policy runs automatically at 2 AM daily. Administrators can view upcoming deletions and, in exceptional cases, manually trigger or defer deletion from the admin panel.

Data deletion is permanent

Once event data is deleted by the retention policy it cannot be recovered. Organisers should export any records they need before the seven-day window closes.
