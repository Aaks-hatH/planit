Guest Invites

Inviting guests & the QR system

Every guest gets a unique invite link, a personal QR code, and optional print-ready materials. No accounts. No apps. No friction.

How invites work

When you add a guest, PlanIt generates a unique 8-character alphanumeric invite code (e.g. AB12CD34). This code is the guest's credential — it identifies them at check-in. No password, account, or app is required.

The invite link takes the form:

https://planitapp.onrender.com/invite/AB12CD34

When a staff member scans the QR code, the scanner reads this URL, extracts the invite code, and looks up the guest record in real-time. The same code can also be entered manually if the QR won't scan.

Treat the invite link like a physical ticket

Anyone with the invite link can present it at check-in. Advise guests to keep their link private, just as they would a concert ticket. If a code is compromised, delete the invite and recreate it — the old code is immediately invalidated.

The guest invite page

When a guest opens their invite link they see a branded, mobile-optimised page showing:

  • Event name, date, time, and location
  • Their name and role badge (Guest / VIP / Speaker)
  • Group size — adults and children in their party
  • Their unique QR code for check-in
  • Table assignment label (if one has been assigned)
  • Any public notes the organiser has added to their record
  • Buttons to save the wallet card or download their print badge
  • An RSVP toggle to confirm or decline attendance

The page works on any device in any modern browser. No installation required.

QR code — /qr/CODE

Each invite has a unique, branded QR code. PlanIt QR codes encode the full invite URL, so any standard QR scanner app can read them — not just the PlanIt camera.

You can embed a guest's QR code as a plain image anywhere:

HTML
<img src="https://planitapp.onrender.com/qr/AB12CD34"
     width="200" alt="Entry QR code" />

The endpoint returns the branded SVG directly — no HTML wrapper. It works as a standard <img> tag in emails, PDFs, and web pages. The image is served with a 24-hour cache header.

Always use width only — let height scale automatically

The QR SVG has a portrait aspect ratio (300 × 430). Setting both a fixed width and height will distort the QR modules and make it unreadable. Always set width only and let the browser scale the height proportionally.

Wallet card — /card/CODE

The wallet card is a full-screen, mobile-optimised page styled like a boarding pass. It shows:

  • Guest name and role badge (colour-coded — amber for VIP, purple for Speaker, grey for standard guests)
  • Event name, date, time, and location
  • Table assignment (if set)
  • QR code, large and high-contrast
  • Share button to copy the card URL to clipboard
https://planitapp.onrender.com/card/AB12CD34

Guests can screenshot the wallet card and save it to their camera roll for offline access at the venue entrance. The dark design ensures the QR is readable even on low-brightness phone screens.

Print badge — /badge/CODE

The badge endpoint serves a print-optimised HTML page sized for A6 paper (148 × 105 mm) and standard Avery label sheets. Open the URL in a browser and press Ctrl+P (or the on-screen Print button) to print.

https://planitapp.onrender.com/badge/AB12CD34

The printed badge includes:

  • Guest name in large, readable type
  • Role badge with colour coding
  • Group size (adults and children)
  • Table assignment label
  • QR code
  • Invite code in monospace text

Print CSS removes all browser chrome, navigation buttons, and on-screen controls. The output is a clean, single-card-per-page badge ready for lanyards or label sheets.

For bulk badge printing, open multiple badge URLs in separate browser tabs and print them in sequence. Browsers handle multi-tab printing well when using the same paper size.

Guest roles

Every guest has one of three roles. Role appears on the invite page, wallet card, print badge, and the boarding-pass review screen at check-in — giving staff at-a-glance context about how to handle each guest.

RoleBadge colourTypical useEffect
GUEST Grey Standard attendees None — informational only
VIP Amber / gold Priority guests, sponsors, executives Prompts staff to provide directed seating, priority entry
SPEAKER Violet Presenters, performers, panellists Prompts staff to direct to speaker area, separate entry lane

Role has no automated access effect — it is purely informational, displayed visually to guide staff decisions at the door. You can use it to train staff to direct different role types to different entry lanes or areas.

Group sizes

An invite can represent a group, not just an individual. When adding a guest, enter the number of adults and children in their party. This information:

  • Appears on the boarding-pass review screen so staff admit the correct headcount
  • Feeds into the Total Admitted counter on the check-in page (sum of adults + children for all admitted groups)
  • Shows on the guest's wallet card and print badge
  • Is counted in the Adults Expected and Children Expected stats

If a group arrives with a different size than recorded, an organiser can edit the invite before or after admission to correct the count. The actual headcount admitted is tracked separately in the analytics.

Security PINs

You can optionally set a security PIN on any invite. If set, staff are required to ask the guest for the PIN and verify it before the admit button becomes active.

The PIN is not shown on the guest's invite page — the guest provides it verbally or in writing. This creates a second factor beyond just having the QR code, which is useful for:

  • High-value paid events where resale or ticket fraud is a concern
  • Guests who need extra identity verification (e.g. corporate events)
  • Restricted areas within a larger event where not all guests have access

PINs are up to 6 characters and can contain letters or numbers. You can set PINs individually or include them in the CSV import column securityPin.

RSVP

Guests can RSVP from their invite page by tapping Attending or Not Attending. Their RSVP status is visible in the guest list on the organiser's check-in dashboard.

RSVP does not block check-in

A guest who RSVPs "Not Attending" can still be admitted at check-in if they show up. RSVP is purely informational — it does not prevent or require admission. Staff always have final say via the boarding-pass admit/deny flow.

Deleting and recreating invites

If an invite code is compromised or shared incorrectly, delete the invite from the guest list. Deletion immediately invalidates the old code — any attempt to scan or look up the old invite code will return "not found". Then recreate the guest with a new invite, which generates a fresh code.

Always warn guests before deleting their invite, as their existing QR code and wallet card will stop working the moment the invite is deleted.
