Guest Invites

Inviting guests & the QR system

Every guest gets a unique invite link, QR code, and optional print-ready badge. No accounts, no apps, no friction.

On this page

How invites work

When you add a guest, PlanIt generates a unique invite code (e.g. AB12CD34). This code is the guest's credential — it is how they are identified at check-in. There is no password, account, or app required.

The invite link takes the form /invite/AB12CD34. Share this link by copying it from the create dialog, or by clicking the email share button to open a pre-filled draft in your mail client.

Treat the link like a ticket

Anyone with the invite link can present it at check-in. Advise guests to keep their link private. If a code is compromised, delete the invite and recreate it for the guest.

The guest invite page

When a guest opens their invite link they see:

  • Event name, date, time, and location
  • Their name and role badge (Guest / VIP / Speaker)
  • Group size (adults and children)
  • Their QR code for check-in
  • Table assignment, if one has been set
  • Any public notes the organiser has added

The page works on any device and in any browser. No app installation is required.

QR code

Each invite has a unique, branded QR code. The QR encodes the guest's invite URL. When scanned at check-in, the scanner extracts the invite code and looks up the guest record.

Direct QR image URL

You can embed a guest's QR code as a plain image anywhere — in an email, PDF, printed material, or a third-party page:

https://planitapp.onrender.com/qr/AB12CD34

This returns the branded QR SVG directly as an image. It works as a standard <img> tag with no wrapper or authentication required.

QR design

PlanIt QR codes use no overlay or centre logo. Overlays reduce scan reliability. The PLANIT branding appears below the QR area, not on top of it.

Wallet card — /card/CODE

The wallet card is a mobile-optimised, full-screen page styled like a boarding pass. It shows the QR code, guest name, event details, role, and table assignment in a dark, high-contrast design.

Guests can screenshot the card and save it to their camera roll so they have it available offline at the venue entrance.

https://planitapp.onrender.com/card/AB12CD34

The card also includes a share button that copies the card URL to the clipboard.

Print badge — /badge/CODE

The badge endpoint serves a print-optimised HTML page designed for A6 paper (148 × 105 mm) or Avery label sheets.

https://planitapp.onrender.com/badge/AB12CD34

Opening this URL and pressing Ctrl+P (or the Print Badge button on screen) prints a clean badge showing:

  • Guest name (large, prominent)
  • Role badge (Guest / VIP / Speaker with colour coding)
  • Group size
  • Table assignment (if set)
  • QR code
  • Invite code in monospace

Print CSS removes all browser chrome and on-screen controls. The result is a clean single card per page.

Guest roles

Every guest has one of three roles. Role is displayed on the invite page, the wallet card, the print badge, and the boarding-pass review screen shown to staff at check-in.

RoleBadge colourTypical use
GUESTGreyStandard attendees
VIPAmber / goldPriority guests who may need different treatment
SPEAKERPurplePresenters, performers, featured guests

Role has no automated effect — it is purely informational for staff at check-in. Use it to prompt staff to give VIPs directed seating or Speakers a separate entry lane.

Security PIN

You can set an optional security PIN on any invite. If set, staff are required to verify the PIN before admitting the guest at check-in. The guest provides the PIN verbally or in writing — it is not shown on their invite page.

Use PINs for high-value events where additional identity verification is required, or for guests who need restricted access.

RSVP

Guests can RSVP from their invite page by confirming or declining attendance. Organisers see RSVP status on the guest list in the check-in dashboard. RSVP does not affect check-in — a guest who RSVPs "no" can still be manually admitted by staff if appropriate.
