Features

Seating Maps

Build a visual drag-and-drop floor plan, assign guests to tables, show staff exactly where to direct each guest at check-in, and sync everything live across devices.

Enabling the seating map

The seating map is available for Enterprise mode events only. To enable it:

  1. 1
    Open the check-in dashboard

    As organiser, click Check-in in the workspace header.

  2. 2
    Click "Set Up Seating"

    The button appears in the check-in toolbar when no seating map exists yet. It opens the seating map editor on first click.

  3. 3
    Build your layout

    The canvas editor loads. You can now add tables, zones, and labels to represent your venue's floor plan.

Once a map exists, the button changes to Seating and is visible to all logged-in users. Staff see a read-only view; organisers see the full editor.

Building your layout

The editor is a canvas-based tool where objects snap to a grid and can be freely positioned and resized. The toolbar on the left contains all available objects.

Placing and editing objects

  • Add an object — click its type in the toolbar, then click the canvas to place it
  • Select — click any object to select it; a highlight border and resize handles appear
  • Move — drag a selected object to reposition it; it snaps to the grid
  • Resize — drag the corner handles of a selected object
  • Label a table — double-click a table to edit its label (e.g. "Table 1", "VIP A", "Press")
  • Set capacity — in the object properties panel, set the maximum number of guests for this table
  • Delete — select an object and press Delete/Backspace, or use the delete button in the properties panel

Assigning guests to tables

There are two ways to assign guests to tables:

  1. Click a table on the canvas to open its detail panel
  2. The panel shows the table's name, capacity, and current assignments
  3. Click Assign Guest — a search field appears
  4. Type the guest's name to search. Click their record to assign them
  5. The guest is now linked to this table. Their invite page and wallet card update immediately to show the table assignment
  1. Open the guest list in the check-in dashboard
  2. Find the guest and click the edit (pencil) icon
  3. In the edit dialog, find the Table field
  4. Type the table name or select from the dropdown (populated from your seating map)
  5. Save. The assignment updates immediately in the seating map, guest list, and the guest's invite page

Check-in integration

When a guest is checked in, their table assignment appears prominently on the boarding pass screen. Staff can immediately tell the guest which table to go to without opening the seating map.

On the seating map itself, tables update their colour to reflect occupancy as guests are checked in:

ColourMeaning
Default (subtle border)Table has assigned guests; none checked in yet
Green fillAll assigned guests have been checked in
Amber fillSome (but not all) assigned guests checked in
GreyTable with no guest assignments

Staff view (read-only)

When staff open the seating map from the check-in page, they see the same canvas as the organiser but cannot move, add, or delete objects. They can:

  • Click any table to see its name, capacity, and list of assigned guests
  • See which guests are checked in vs still pending at each table
  • Use the map to orient themselves in the venue and direct guests to the correct table

The staff view is useful to leave open on a tablet at a concierge station near the entrance — staff can quickly look up table assignments when guests ask.

Real-time sync

All seating map changes sync instantly to all connected devices via Socket.IO. When the organiser:

  • Moves a table
  • Assigns a guest to a table
  • Changes a table label or capacity
  • Adds or removes objects

…every staff device and every browser with the seating map open receives the updated layout within milliseconds. Staff never work from a stale map.

Layout design tips

  • Build the map before adding guests — finalize your floor plan layout, then assign tables. Changing table names after assigning guests can cause guest records to lose their assignment reference.
  • Use clear, short table labels — "T1", "VIP-A", "HEAD" are easier for staff to relay verbally than "Corporate Table Seven"
  • Include orientation markers — add an Entrance object so staff know which way is front. Add labels for "Stage" and "Bar" so the map matches what staff see in the room
  • Set capacities accurately — the map shows how full each table is (e.g. 6/8 guests). Correct capacities make it easier to see at a glance if a table is overbooked
  • Assign all guests before event day — last-minute table changes are possible but stressful. Complete assignments the day before and do a walkthrough with staff
