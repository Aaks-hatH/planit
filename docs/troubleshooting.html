Troubleshooting

Step-by-step fixes for the most common issues — loading errors, scanner problems, password lockouts, offline failures, and walkie-talkie faults.

The app shows "Service Crash" or won't load

This is usually a cold start, not a real crash

PlanIt's backend spins down after 15 minutes of inactivity and takes 30–60 seconds to restart. Wait 60 seconds and refresh before escalating.

  1. 1
    Wait 60 seconds and hard-refresh

    Press Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac) to bypass the browser cache and reload. On mobile, pull down to refresh or close and reopen the tab.

  2. 2
    Check the status page

    Open planitapp.onrender.com/status from a different device or browser to see if there is an active incident. If all systems are green, the issue may be local.

  3. 3
    Try a different network

    Switch from Wi-Fi to mobile data (or vice versa). Some corporate or venue Wi-Fi networks block WebSocket connections, which prevents PlanIt from functioning correctly.

  4. 4
    Try a different browser

    If Chrome is failing, try Safari or Edge. Browser extension conflicts (ad blockers, VPNs) can interfere with API requests.

  5. 5
    Disable browser extensions

    Temporarily disable extensions — especially ad blockers, privacy shields, or VPN extensions — which can block fetch requests or WebSocket connections.

  6. 6
    Still broken?

    Submit a report at /support with your browser version, device, and a description of what you see.

The app is very slow to load

PlanIt on Render's free tier has limited compute resources. During periods of high load:

  • API response times may increase to 2–5 seconds instead of the usual <500ms
  • Real-time events (chat messages, check-in updates) may lag by 1–3 seconds
  • File uploads may take longer than usual

These are infrastructure constraints, not bugs. If the slowness persists beyond 10 minutes, check the status page for any ongoing incidents.

QR scanner won't start or camera is black

  1. 1
    Check camera permission

    Look in the browser address bar for a camera or lock icon. Click it and set Camera to Allow. Reload the page after changing permission.

  2. 2
    iOS-specific: check per-app camera permission

    Go to Settings → Safari (or Chrome) → Camera and set it to Allow. On newer iOS: Settings → Privacy → Camera → find your browser and enable it.

  3. 3
    Android-specific: check site permissions

    In Chrome, tap the lock icon next to the URL → Site settings → Camera → set to Allow.

  4. 4
    Make sure no other app has the camera

    Close any open video call apps (FaceTime, Zoom, Teams, Google Meet). Only one app can use the camera at a time.

  5. 5
    Reload the page

    The scanner initialises on page load. A permission change requires a reload to take effect.

  6. 6
    Try a different browser

    Chrome (Android) and Safari (iOS) have the most reliable scanner implementations. Firefox and Samsung Internet sometimes have camera initialisation delays.

Offline check-in not working

Offline mode requires the check-in page to have been fully loaded while online. If the page was opened for the first time without a connection, the cache was never populated.

  1. 1
    Verify the page loaded online

    The guest list must have been fetched at least once with a working internet connection. If you opened the check-in page and it showed a loading spinner that never resolved, the cache is empty.

  2. 2
    Check for the offline indicator

    When PlanIt detects it has lost connectivity, it shows an offline banner at the top of the check-in screen. If you don't see this banner but scans are failing, connectivity may be intermittent rather than fully offline.

  3. 3
    Wait for sync on reconnect

    When connectivity returns, queued offline check-ins sync automatically. You will see a sync status indicator while they upload. Do not close the page during sync.

  4. 4
    Prevention for next time

    Load the check-in page and wait for the full guest list to appear at least 30 minutes before the event, while you still have good connectivity. This seeds the offline cache.

Locked out — forgot organiser password

There is no self-service password reset. If you have lost your organiser (account) password:

  1. 1
    Check your password manager

    If you use a password manager (1Password, Bitwarden, LastPass, iCloud Keychain, browser saved passwords), search there first.

  2. 2
    Check your other browsers

    If you used a different browser to create the event, check that browser's saved passwords.

  3. 3
    Contact support

    Submit a request at /support. Include the event name, slug, and approximate creation date. If you can prove ownership (e.g. you created the event from a specific device), recovery may be possible.

For your next event

Save your organiser password in a password manager the moment you create the event. Before sharing the event link, confirm you can successfully log in as organiser from a fresh incognito window.

Real-time updates stopped working

If check-in counts, chat messages, or task changes are no longer updating live:

  1. 1
    Check your connection

    The WebSocket connection may have dropped. Look for a connection status indicator in the workspace header. A disconnected state means real-time is paused.

  2. 2
    Reload the page

    A page reload re-establishes the WebSocket connection and fetches the latest state from the server.

  3. 3
    Check if the network blocks WebSockets

    Some corporate or hotel Wi-Fi networks block WebSocket connections. Switch to mobile data to test. PlanIt falls back to HTTP long-polling if WebSockets are unavailable, but this is slower and may not work on all restricted networks.

Wallet card not adding correctly

  • iOS: Wallet pass requires iOS 6+ and the Wallet app installed (it is pre-installed on all iPhones). If the "Add to Wallet" button does nothing, check that Wallet is not restricted under Screen Time → Content & Privacy Restrictions.
  • Android: Google Wallet must be installed and set up. If the pass doesn't add, ensure Google Wallet is updated to the latest version from the Play Store.
  • Details not updating: Wallet passes are cached on-device. If you change the event date or table assignment after a guest has added their pass, the update may take up to 24 hours to push to their device. Ask guests to open the invite page and re-add the pass if they need the latest information immediately.

Still stuck?

If none of the above resolves your issue, submit a support request at planitapp.onrender.com/support. Include:

  • Browser name and version (e.g. "Chrome 124 on Android 14")
  • Device type (iPhone 15, Samsung Galaxy S23, MacBook Air M2, etc.)
  • The exact error message or behaviour you saw
  • What you were trying to do when it happened
  • Any steps you already tried from this page
